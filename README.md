# FC 강화 시뮬레이터

FC 온라인(EA SPORTS FC ONLINE) 선수 강화를 모바일에서 간편하게 연습할 수 있는 PWA 시뮬레이터.
선수 사진/식별 없이 OVR과 강화 단계만으로 동작한다.

## 사용법

1. 강화 대상의 현재 OVR과 강화 단계를 선택한다.
2. 강화 재료를 최대 5개까지 OVR/비용으로 입력한다.
3. 필요하면 강화등급 보호를 켜고 비용을 입력한다.
4. "강화 시도"를 누르면 성공/실패가 결정되고, 비용과 기록이 누적된다.

## 확률 데이터 출처

기본 강화 성공 확률(부스트 게이지 100% 기준)과 단계별 누적 OVR은 넥슨 공식 공지
"(26.4 기준) 선수 강화 성공 확률, 강화 실패 시 복구 확률 및 크리티컬 확률 안내"와
FC 온라인 인벤 동일 자료로 교차 확인했다 (`src/enhanceData.js` 참고).

재료 OVR을 강화부스트 게이지(%)로 환산하는 계산은 실측 자료
(["강화 재료표"](https://canfactory.tistory.com/901), 재료 OVR과 대상 OVR 차이 × 강화 단계별
포인트, 풀게이지 = 5포인트)를 그대로 테이블화했다 (`src/materialPoints.js`). 8강 이후 단계는
공식 자료에 없어 기본적으로 "7강→8강" 값을 재사용하지만, 11강·12강에서 실제 게임 화면으로
직접 확인한 값(재료 OVR 차이 0 → 1.00포인트, +3 → 약 2.465포인트, +4 → 3.325포인트)은 8강
이후 전체 구간에 덮어써서 반영했다 (두 단계에서 거의 동일한 값으로 재확인되어, 7강 이후로는
값이 더 변하지 않고 수렴한다고 판단).

원본 표에서 재료가 대상보다 3 이상 낮은 칸("-")은 측정이 안 된 것일 뿐 실제로 0은 아니다.
표 전체를 분석해보면 재료 OVR이 1 낮아질 때마다 포인트가 거의 정확히 0.75배가 되는 패턴이
모든 단계에서 일관되게 나타나므로(예: +1,0,-1,-2 = 1.33→1.00→0.75→0.57), 측정값이 없는
칸은 0 대신 가장 가까운 상위 diff 값에 0.75를 거듭 곱해 추정한다 (0이 되는 게 아니라 점점
작아질 뿐).

"동일 선수" 재료 투입 시 발동하는 크리티컬 강화(+2등급, 1~4강 한정)와 복구는 선수 식별이
필요해 이 단순 모델에는 포함하지 않았다. 강화등급 보호는 선수 식별이 필요 없는 별도 시스템이라
포함되어 있다 (실패해도 등급 유지, 보호 미사용 시 실패하면 1강으로 초기화).

## 개발

```bash
npm install
npm run dev      # http://localhost:5173/fifa-simulator/
npm run build
```

`main` 브랜치에 푸시하면 GitHub Actions가 GitHub Pages로 자동 배포한다
(`.github/workflows/deploy.yml`). 저장소 Settings → Pages → Source를
"GitHub Actions"로 설정해야 한다.
